# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

This is **not a software codebase**. It is a content/publishing repo that produces a French-language e-book, *« Devenir un dieu du Prompt Engineering »* (a pedagogical manual on prompt engineering), as a print-ready A4 PDF.

All work lives in the `Dieu du prompt/` directory. The **single source of truth** is the Markdown file; everything else (PDF) is a generated artifact.

## Files that matter

- `Dieu du prompt/Devenir Dieu du Prompt Engineering - version simplifiee.md` — **the manuscript.** Edit this; never edit the PDF directly.
- `Dieu du prompt/style.tex` — **the print stylesheet** (LaTeX preamble injected by `build.sh`). Controls the brown/gold palette `#c8880a` / `#5c3d00`, the per-page gold frame, the light "quadrillage" background grid, fonts (Palatino body / Nimbus Sans headings / DejaVu mono, 13pt), heading styles, code/quote boxes, and the auto-generated table of contents. Edit visual styling here.
- `Dieu du prompt/frontmatter.tex` — LaTeX inserted right after `\begin{document}`: the full-bleed cover page and the title block atop the first content page.
- `Dieu du prompt/couverture.png` — cover image (rendered full-bleed as page 1).
- `Dieu du prompt/build.sh` — the build pipeline (see below).
- `Dieu du prompt/Devenir Dieu du Prompt Engineering.pdf` — committed final output, regenerated by `build.sh`.
- `Dieu du prompt/style.css` — **legacy**, from the old HTML/wkhtmltopdf pipeline. No longer used by `build.sh`; kept for reference only.

## Build pipeline

`build.sh` produces the PDF via **LaTeX**: **pandoc** (Markdown → LaTeX, driving `style.tex` + `frontmatter.tex`) → **lualatex** (run twice, so the table of contents resolves its page numbers). It works out of `/tmp/livre`, which it creates and populates itself (no manual setup needed).

```bash
cd "Dieu du prompt" && ./build.sh
```

`lualatex` (not pdflatex/xelatex) is required: the build uses `fontspec` system fonts and a `luaotfload` fallback so the manuscript's emojis render in colour via **Noto Color Emoji**.

The Markdown's `## Sommaire` block is raw HTML and is **dropped** during preprocessing — `build.sh` replaces it with a real `\tableofcontents` (depth 1, so only `## N.` chapters appear, with auto page numbers). The H1 title and `### Édition enrichie` subtitle lines are also stripped and re-rendered by `frontmatter.tex`. On a build failure, `build.sh` prints the tail of `/tmp/livre/lualatex.log`.

Requires on PATH: `pandoc`, `lualatex` + LaTeX packages (`texlive-luatex`, `texlive-latex-extra`, `texlive-pictures`, `texlive-fonts-recommended`, `texlive-lang-french`), the `fonts-noto-color-emoji` font, and `pdfinfo` (poppler-utils).

## Authoring conventions

- **Language is French.** Match the existing register: warm, second-person (`tu`), pedagogical, occasional emoji and lived-experience anecdotes. Use numbered `## N. Titre` sections — the table of contents is now generated automatically from these headings, so you no longer maintain it by hand. (The old `## Sommaire` HTML block still sits in the Markdown but is dropped at build time; it can be left as-is or removed.) Each `## N.` chapter starts on a fresh page.
- **Manual page breaks:** insert `[[PAGE - X]]` on its own line. `build.sh` rewrites any `[[PAGE - ...]]` marker into a LaTeX `\newpage` before conversion. (Currently none are used; the mechanism exists.)
- **Avoid AI-writing tells.** The manuscript has been deliberately "humanized" (see git history). When drafting or editing prose, keep it human: vary sentence length, cut hype/filler, no em-dash overuse, no three-part-list reflex. The `humanizer-zh` skill under `.agents/skills/` is installed for this purpose.

## Workflow

After editing the Markdown, rebuild the PDF and commit both the `.md` and the regenerated `.pdf` together. Commit messages in this repo are written in French.
